개발자: 하늘 역할: 기획 · 백엔드 · 프론트 형태: 개인 사이드 / 포트폴리오

왜 만들었을까 ?

주류 커뮤니티는 많은데, “내 취향”인 주류 커뮤니티는 없더라 (그리고 술 좋아함 😘)

첫 버전 목표는 ‘가볍고 빠른 SSR 경험’ 검증. 카드/보드/후기를 한 뷰에 모으고 로그인 없이 흐름을 체험하게 설계.

뭘 만들었을까 ?

  • Thymeleaf 서버 바인딩 → 첫 화면 즉시 노출
  • 도메인 단순화(카드·게시판·후기)
  • 검색/정렬/무한스크롤은 Fetch로 점진 확장

그리고… 포트폴리오 만드는 것도 가끔 힘들다! 😮‍💨

역할

  • 요구 정의 · 화면 설계 · 라우팅 구조
  • 도메인/엔티티 초안, 컨트롤러/뷰 연결, 타임리프 바인딩
  • 정적 리소스 구조(css/img/js) 및 빌드/실행 흐름 정리

요즘은 코드 정리까지 하느라 하루가 훅 간다 ,,, 🧹

기술 스택

Spring Boot Thymeleaf(SSR) JPA MySQL HTML/CSS/JS

회고 / 배운 점

  • SSR로 초기 표시 빨라지고, 점진적 CSR로 상호작용 확장
  • JPA N+1 쿼리/지연 로딩 문제를 페치 조인으로 해결
  • 컴포넌트/레이아웃 분리로 타임리프 템플릿 재사용성↑

성능/렌더링 — 첫 화면을 SSR로 그리고, 이후 목록/정렬은 Fetch로 점진 로딩. 불필요한 DOM 업데이트 줄이니 체감 속도 좋아짐.

데이터 접근 — 리스트 뷰에서 N+1 발생 → fetch join으로 해결. DTO 투입 구간과 엔티티 노출 구간을 분리해서 응답 안정화.

템플릿 구조 — 레이아웃/프래그먼트로 공통 헤더/카드 컴포넌트화. 부분 변경이 쉬워지고 중복 제거됨.

UX 디테일 — “요약 → 팝업 디테일” 패턴으로 정보 밀도 조절. 모바일에서 터치 영역 키우고 포커스 스타일 정리.

아쉬운 점/다음 — 이미지 업로드는 저장소 선택 이슈로 보류. 이후 S3/클라우드 스토리지 + 썸네일 파이프라인 검토 예정.
