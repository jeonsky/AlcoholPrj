개발자: 하늘 역할: 기획 · 백엔드 · 프론트 형태: 개인 사이드 / 포트폴리오

왜 만들었나

주류 커뮤니티는 많은데, “내 취향”인 주류 커뮤니티는 없더라 (그리고 술 좋아함 😘)

첫 버전 목표는 ‘가볍고 빠른 SSR 경험’ 검증. 카드/보드/후기를 한 뷰에 모으고 로그인 없이 흐름을 체험하게 설계.

무엇으로 해결했나

  • Thymeleaf 서버 바인딩 → 첫 화면 즉시 노출
  • 도메인 단순화(카드·게시판·후기)
  • 검색/정렬/무한스크롤은 Fetch로 점진 확장

그리고… 포트폴리오 만드는 것도 가끔 힘들다! 😮‍💨

내 역할 / 기여

  • 요구정의 · 화면 설계 · 라우팅 구조
  • 도메인/엔티티 초안, 컨트롤러/뷰 연결, 타임리프 바인딩
  • 정적 리소스 구조(css/img/js) 및 빌드/실행 흐름 정리

요즘은 최적화/정리까지 하느라 하루가 훅 간다… 🧹

기술 스택

Spring Boot Thymeleaf(SSR) JPA MySQL HTML/CSS/JS

초기엔 SSR로 빠르게, 상호작용은 부분 CSR로 점진 확장 💡

로드맵

  • 게시판/후기 페이징·정렬·검색
  • 파일 업로드(프로필/이미지)
  • 로그인/권한에 따른 메뉴 제어

알파→베타 전환하며 알림/프로필 커스텀도 추가 예정!
